XmlConverter

Entwicklung des Programms „XmlConverter“. Das C#-Programm liest aus einer sogenannten OPC UA XML-Datei“ die Namen und Adressen von Knoten (Nodes) und fügt sie in eine Excel-Tabelle ein. Zudem liest das Programm die Datentypen von Knoten. Die Datentypen können beliebig tief verschachtelt sein. In einer OPC UA XML-Datei ist der sogenannte Adressraum eines OPC UA-Servers beschrieben, der sich zum Beispiel aus Knoten zusammensetzt, über die Daten aus Industriesteuerungen gelesen werden können. Der XmlConverter vereinfacht den Umgang mit OPC UA XML-Dateien, indem er die wesentlichen Informationen für OPC UA-Clients zur Verfügung stellt. Der XmlConverter wird zum Beispiel von System-Integratoren verwendet, die ein SCADA-System an eine Industriesteuerung anschließen wollen. Das Programm ist eine Microsoft WPF-Anwendung.Die Anwendung arbeitet mit merhreren Threads und rekursiven Methoden. Ich habe den XmlConverter auch als ASP-NET-Webseite entwickelt.

Sender und Empfänger für eine Testumgebung

Entwicklung von zwei C#-Programmen: ein Sender, der Daten in ein LAN (Ethernet) einspeist und ein Empfänger, der die Daten entgegen nimmt. Die beiden Programme dienen dazu, für den Test eines anderen Programms eine Testumgebung zu erzeugen. Das zu testende Programm begrenzt den Datenstrom, der von einem Windows-Rechner ausgesendet werden darf. Deshalb ist es erforderlich, eine Testumgebung aufzubauen, in der das zu testende Programm sinnvoll betrieben werden kann.
